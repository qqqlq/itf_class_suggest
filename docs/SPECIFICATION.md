# システム仕様書 (System Specification)

## 1. 概要
筑波大学情報メディア創成学類 (MAST) の学生を主に対象とした、履修計画サジェスト・卒業要件確認システム。
TWINS からエクスポートされた成績データ (`SIRSxxxx.csv`) を元に、卒業要件の充足状況を判定し、次年度の最適な履修科目を提案する。

## 2. データの構造と連携

### 2.1 KdB データ (筑波大学シラバス)
- **ソース**: `data/courses/kdb_dict.json`
- **役割**: 2026年度科目マスタ (`2026.json`) の補完。
- **補完項目**: 
  - 科目名の日本語化（英語名からの自動変換）
  - 正確な標準履修年次（1~4年）の取得
  - 単位数、開講期の情報の取得
- **連携ロジック (`lib/kdbEnricher.ts`)**:
  - `getDisplayName()`: IDから日本語名を優先取得。
  - `getStandardYear()`: 文字列形式の年次（例: "3・4"）から下限値を取得。
  - `findCourseIdByName()`: 日本語科目名からの ID 逆引き（GC必修科目の特定に使用）。

### 2.2 成績パース (`lib/csvParser.ts`)
- TWINS 成績 CSV のヘッダー構造を認識し、各行を `GradeRecord` 型に変換。
- パース時に KdB 辞書と照合し、取得済み科目にメタデータ（標準年次等）を付与。

## 3. 卒業要件の管理 (`lib/requirementChecker.ts`)

- **カリキュラム定義**: `data/curricula/mast.json`
- **階層構造**: 
  1. **グループ (Group)**: 専門/専門基礎/基礎(共通)/基礎(関連)
  2. **カテゴリ (Category)**: 各グループ内の詳細区分（必修/選択等）
- **判定ロジック**:
  - `required`: ID または科目名（KdB逆引き含む）での完全一致判定。
  - `elective`: プレフィックス (例: `GC2`) による前方一致判定。
  - 判定順序: 必修 (Required) → 選択 (Elective) → 自由 (Free/Unclassified) の順で、重複カウントを避けながら単位を配分。

## 4. 履修サジェストアルゴリズム (`lib/courseSuggester.ts`)

4つのフェーズで科目を抽出する：

1. **Phase 1: 必修・取り逃がし**: 
   - 未取得の必修科目のうち、現在または過去の年次配当のものを最優先。
2. **Phase 2: 再履修**: 
   - 直近で「D評価」となった科目を優先的に追加。
3. **Phase 3: カテゴリ充足**: 
   - 卒業要件で不足している単位を埋めるため、該当カテゴリの科目を追加（GA/GC系優先）。
4. **Phase 4: 単位数確保**: 
   - 年間目標（デフォルト40単位）に達するまで、当該年次の科目を追加。

### 重複・取得済み判定の高度化
- **同一科目・別クラスの考慮**: 取得済みの科目番号（ID）だけでなく、**KdB 日本語名**が一致する科目も「取得済み」として判定する。
  - 例: `GA15131` (取得済み) があれば、別クラスの `GA15111` (提案候補) は自動的に除外される。
- **提案リスト内の重複排除**: 同じ日本語名を持つ科目が複数候補にある場合、学類向けコード (`GC`) を最優先し、次に学群共通コード (`GA`) を優先して 1 件に絞り込む。

### 時間割の衝突回避 (`lib/timetableResolver.ts`)
- 各フェーズで科目を追加する際、既にサジェストされた科目と曜日・時限・モジュールが重複しないか検証。

## 5. ユーザーインターフェース

### 5.1 卒業要件表 (`components/GraduationTableView.tsx`)
- マスタ内の全てのカテゴリを一覧表示。
- 文字色・背景色によるステータス表示：
  - **青/緑**: 取得済み。
  - **赤**: 未取得の必修（要履修）。
  - **黄色**: 警告（当該年次までに取るべき科目）。

### 5.2 時間割グリッド (`components/TimetableGrid.tsx`)
- `mihon.jpg` をベースにした高視認性グリッド。
- 春A/B/C, 秋A/B/C のモジュール切替に対応。
- 優先度に応じたカラーカード表示。
- グリッド外科目（随時・集中等）の別枠リスト表示。

## 6. 開発ガイドライン
- **スタイリング**: Vanilla CSS + `globals.css` 変数。
- **データ更新**: KdB の生データを更新する際は `scripts/generate-kdb-dict.ts` を実行して `kdb_dict.json` を再生成すること。
